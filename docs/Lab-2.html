Framing the Problem¶

Goal:¶

  • The goal of this task is to build a predictive model that helps physicians identify patients at risk of diabetes progression based on their physical and medical characteristics. Specifically, we want to predict the disease progression one year after baseline using the provided features.

Problem Framing:¶

  • Type of Problem: This is a regression problem since we are predicting a continuous target variable (progression, which represents disease progression one year after baseline).
  • Target Variable: The dependent variable (output) we want to predict is progression, which measures the severity of diabetes progression.

In [12]:
def fit_poly(degree, X_train, y_train, X_val, y_val):
    poly = PolynomialFeatures(degree=degree)
    X_train_poly = poly.fit_transform(X_train[['bmi']])
    X_val_poly = poly.transform(X_val[['bmi']])
    
    model = LinearRegression()
    model.fit(X_train_poly, y_train)
    
    y_train_pred = model.predict(X_train_poly)
    y_val_pred = model.predict(X_val_poly)
    
    train_r2 = r2_score(y_train, y_train_pred)
    val_r2 = r2_score(y_val, y_val_pred)
    train_mae = mean_absolute_error(y_train, y_train_pred)
    val_mae = mean_absolute_error(y_val, y_val_pred)
    train_mape = mean_absolute_percentage_error(y_train, y_train_pred)
    val_mape = mean_absolute_percentage_error(y_val, y_val_pred)
    
    return {
        'degree': degree,
        'train_r2': train_r2, 'val_r2': val_r2,
        'train_mae': train_mae, 'val_mae': val_mae,
        'train_mape': train_mape, 'val_mape': val_mape
    }

How many trainable parameters are we fitting for each of the models?¶

  • The number of trainable parameters grows with the degree of the polynomial because we are fitting more complex models (higher degrees add more features derived from BMI).
  • Intercept: Every polynomial model has an intercept (constant term).
  • Coefficients: The number of coefficients depends on the degree of the polynomial.
  • Degree 0: 1 parameter (just the intercept).
  • Degree 1: 2 parameters (intercept + BMI coefficient).
  • Degree 2: 3 parameters (intercept, BMI, BMI²).
  • Degree 3: 4 parameters (intercept, BMI, BMI², BMI³).
  • Degree 4: 5 parameters.
  • Degree 5: 6 parameters.

Conclusion¶

Model Performance:

  • The degree 1 (linear) model was chosen as the best model due to its simplicity and reasonable performance.
  • On the test set, the model achieved:
    • R-squared (R²): 0.281, indicating that the model explains 28.1% of the variance in disease progression.
    • Mean Absolute Error (MAE): 51.51, suggesting that the model's predictions are, on average, off by about 51 units of disease progression.
    • Mean Absolute Percentage Error (MAPE): 43.17%, meaning that the model's predictions deviate by about 43% from the actual values.

Model Equation: Disease Progression=152.08+975.28⋅BMI

Model Limitations:

  • R² = 0.281: The model explains only 28.1% of the variation in disease progression, suggesting that other factors (beyond BMI) are influencing the outcome.
  • While the linear model was chosen for simplicity, higher-degree polynomial models may have captured more complex relationships between BMI and disease progression.
  • Using higher-degree polynomial models could introduce overfitting, especially with smaller datasets, so the simplicity of the degree 1 model is beneficial in this case.

Conclusion:

  • The linear model demonstrates a clear positive relationship between BMI and disease progression, making it a simple and interpretable tool for predicting disease progression. However, the model's modest R² score highlights the need to include more features to improve prediction accuracy and better understand the factors contributing to diabetes progression.
